Tri par Fusion¶

John von Neumann, 1945

Principe¶

On peut facilement fusionner deux listes tries en une seule en en extrayant itrativement le plus petit lment. Celui-ci est forcment aussi le plus petit de l'une des deux listes fusionner.

Ce procd est appel fusion et est au cur de l'algorithme de tri par fusion rcursif.

  • Si le tableau n'a qu'un lment, il est dj tri.
  • Sinon, sparer le tableau en deux parties peu prs gales.
  • Trier rcursivement les deux parties avec l'algorithme du tri fusion.
  • Fusionner les deux parties tries en un seul tableau tri.

Fusion¶

Entre: les sous-tableaux T[permier:limite] et T[limite:dernier], supposs tris.

Sortie: le tableau T[premier:dernier] est tri

def fusion(T, premier, limite, dernier):     
    asd1.affiche_entree_fusion(T,premier, limite, dernier)

    T1 = T[premier:limite].copy()
    T2 = T[limite:dernier].copy()
    
    i = premier; i1 = i2 = 0
    
    while i1 < len(T1) and i2 < len(T2):
        if T2[i2] < T1[i1]:
            T[i] = T2[i2]; i2 += 1
        else:
            T[i] = T1[i1]; i1 += 1
        i += 1

    for j in range(i1,len(T1)): T[i] = T1[j]; i += 1

    for j in range(i2,len(T2)): T[i] = T2[j]; i += 1

    asd1.affiche_sortie_fusion(T1,T2,T[premier:dernier])

Récursion¶

  • Entre: le tableau T[premier:dernier] dans un ordre quelconque

  • Cas trivial: T a 0 ou 1 lment, ne rien faire

  • Cas gnral: diviser T en deux, les trier rcursivement, puis les fusionner

  • Sortie: T[premier:dernier] est tri

def recursion(T,premier,dernier):  
    asd1.affiche_entree_tri_fusion(T,premier, dernier)
    
    N = dernier - premier
    if N >= 2:
            
        milieu = premier + int(N/2)
        recursion(T,premier,milieu)
        recursion(T,milieu,dernier)
        fusion(T,premier,milieu,dernier)
        
def tri(T):
    recursion(T,0,len(T))

En résumé¶

Le tri par fusion est mis en oeuvre de manire rcursive. Il commence par faire deux appels rcursifs sur les moitis gauche et droite du tableau, puis les fusionne.

In [6]:
def fusionner(T,premier,milieu,dernier,comparer = asd1.plus_petit):        
        T1 = T[premier:milieu].copy(); i1 = 0
        T2 = T[milieu:dernier].copy(); i2 = 0
        i = premier; 
    
        while i1 < len(T1) and i2 < len(T2):
            if comparer(T2[i2],T1[i1]):
                T[i] = asd1.assigner(T2[i2]); i2 += 1;
            else:
                T[i] = asd1.assigner(T1[i1]); i1 += 1;
            i += 1
        for j in range(i1,len(T1)):
            T[i] = asd1.assigner(T1[j]); i += 1; 
        for j in range(i2,len(T2)):
            T[i] = asd1.assigner(T2[j]); i += 1;
                
def tri_fusion_recursif(T,premier,dernier,comparer = asd1.plus_petit): 
    if dernier - premier >= 2:          
        milieu = premier + int((dernier - premier)/2)
        tri_fusion_recursif(T, premier, milieu, comparer)
        tri_fusion_recursif(T, milieu, dernier, comparer)
        fusionner(T,premier,milieu,dernier,comparer)
        
def tri_fusion(T, comparer = asd1.plus_petit ):
    tri_fusion_recursif(T,0,len(T),comparer)

 Complexité¶

Evaluons d'abord la complexit du tri d'un tableau au contenu gnr alatoirement.

In [7]:
asd1.evalue_complexite(tri_fusion, asd1.tableau_aleatoire, 
                       "tri fusion, tableau alatoire")

La complexit du tri est linarithmique en $\Theta(n\log(n))$. C'est assez simple dmontrer si l'on considre l'algorithme par niveau de rcursion.

  • Notons tout d'abord que chaque appel rcursif divise par deux la taille du tableau traiter. La profondeur totale de la rcursion est donc proportionnelle $\log_2(n)$.
  • Considrons maintenant une profondeur de rcursion donne.
    • au premier niveau, la fusion est effectue une seule fois sur l'ensemble des $n$ lments
    • au second niveau, deux fusions sont effectues, mais chaque fois sur la moiti des lments seulement, pour un total de $2 \frac{n}{2} = n$ lments fusionns
    • au $k^{ieme}$ niveau, environ $2^{k-1}$ fusions sont effectues, chacune sur environ $\frac{n}{2^{k-1}}$ lments pour un total de n lments fusionns.
    • chaque niveau de rcursion, la fusion requiert donc en tout de dplacer $n$ lments
  • Il y a donc en tout $n\log(n)$ lments tests et dplacs par les oprations de fusion

Stabilité¶

Le tri fusion est stable.

La ligne critique est le test T2[i2] < T1[i1]. En cas d'galit entre l'lment le plus petit de T1 ou de T2, il faut d'abord copier dans T celui de T1, puisque celui-ci vient de la section [premier:milieu] qui est antrieure la section [milieu:dernier]

Vrifions le en triant par parties fractionnaires puis par parties entires.
