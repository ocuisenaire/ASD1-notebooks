Tri par insertion¶

Principe¶

Pour un tableau de $n$ lments, le tri par insertion

  • parcourt le tableau du $2^{ieme}$ au $n^{ieme}$ lment.
  • insre l'lment courant $k$ dans la partie du tableau dj trie $[0:k-1]$

Pour insrer, il dplace d'une position vers la droite tous les lments strictement plus grands que l'lment insrer, ce qui libre une position pour l'insertion.

Boucle interne¶

Le coeur de l'algorithme consiste insrer l'lment d'indice $k$ dans le tableau suppos dj tri des lments d'indices $0$ $k-1$.

  • copier l'lment k dans une variable temporaire tmp
  • tant que tmp est plus petit que l'lment prcdent
    • dplacer cet lment vers la droite
    • reculer d'une position dans le tableau
  • crire tmp dans l'emplacement libr.
In [1]:
def inserer_un_element(T,k):
    tmp = T[k]
    i = k
    while i > 0 and tmp < T[i-1]: 
        T[i] = T[i-1]    
        i -= 1 
    T[i] = tmp

Un tableau de 1 lment est toujours tri.

On commence donc par insrer le deuxime lment (d'indice 1) dans le tableau ne contenant que le premier lment

En résumé¶

Le tri par insertion effectue deux boucles imbiques.

  • La boucle interne insre l'lment d'indice k dans le sous-tableau le prcdant.
  • La boucle externe fait varier cet indice k de la deuxime la dernire position.
In [11]:
def tri_par_insertion(T, comparer = asd1.plus_petit):
    N = len(T)
    for k in range(1,N):
        tmp = T[k]       
        i = k
        while i > 0 and comparer(tmp,T[i-1]):
            T[i] = asd1.assigner(T[i-1])
            i -= 1
        T[i] = asd1.assigner(tmp)

Complexité¶

Pour valuer la complexit de cet algorithme, valuons d'abord la complexit du tri d'un tableau au contenu gnr alatoirement.

In [13]:
asd1.evalue_complexite(tri_par_insertion, 
                       asd1.tableau_aleatoire, 
                       "tri par insertion, tableau alatoire")

Notons que

  • le nombre de comparaisons et d'critures est quasiment gal (*).

  • leur complexit est d'ordre quadratique en $\Theta(n^2)$ pour trier $n$ lments.

  • le nombre exact de comparaisons varie, sans doute en fonction du contenu du tableau

(*) La seule diffrence provient des rares fois ou la boucle while s'arrte sur le test i>0 et ne teste pas comparer(tmp,T[i-1]) par court-circuit

Vrifions cette dernire hypothse en triant un tableau dj tri

In [14]:
asd1.evalue_complexite(tri_par_insertion, asd1.tableau_trie, 
                       "tri par insertion, tableau tri")

La complexit est linaire en $\Theta(n)$. Le test comparer(tmp,T[i-1]) renvoye toujours False. C'est le meilleur cas pour le tri par insertion.

Observons maintenant le cas inverse d'une entre trie l'envers

In [15]:
asd1.evalue_complexite(tri_par_insertion, 
                       asd1.tableau_trie_inverse, 
                       "tri par insertion, tableau invers")

La complexit est ici quadratique en $\Theta(n^2)$. C'est le pire cas.

Regardons enfin un cas d'importance pratique, celui d'un tableau presque tri. Ecrivons d'abord une fonction gnrant un tel tableau.

La complexit est ici approximativement linaire en $\Theta(n)$.

La fonction de gnration de tableau presque tri utilise ici garantit qu'aucun lment n'est plus de 5 places de sa position finale.

La boucle interne itre au maximum 5 fois.

Le tri par insertion est remarquablement efficace pour trier un tableau presque tri.

A quel point il est efficace en pratique dpend videmment de ce que l'on entend par presque

Stabilité¶

Le tri par insertion est stable.

En effet, le test tmp < T[i-1] garantit qu'on ne dplace pas vers la droite un lment gal celui que l'on cherche insrer. L'ordre des lments gaux est donc prserv.

Vrifions le graphiquement en triant par parties fractionnaires puis entires.

In [19]:
asd1.test_stabilite(tri_par_insertion)

Le tri est stable

Notons que le test strict tmp < T[i-1] est essentiel la stabilit. Remplacer le test tmp <= T[i-1], ou not( T[i-1] < tmp ) rend le tri instable

In [20]:
def tri_par_insertion_errone(T,plus_petit):
    N = len(T)
    for k in range(1,N):
        tmp = T[k]
        i = k
        while i > 0 and not plus_petit(T[i-1],tmp):
            T[i] = T[i-1]
            i -= 1
        T[i] = tmp
