Arbres AVL¶

G. Adelson-Velskii et E. M. Landis, An Algorithm for the Organization of Information. Soviet Mathematics Doklady, 3:12591263, 1962

Principe¶

Les arbres AVL sont des arbres binaires de recherche qui garantissent qu'aucun noeud de l'arbre n'a un dsquilibre en dehors de l'intervalle [-1,1].

Le dsquilibre d'un noeud est la diffrence de hauteur entre ses sous-arbres gauches et droits.

Pour les mettre en oeuvre, il faut - chaque insertion et suppression -

  • mesurer le dsquilibre de chaque noeud travers
  • corriger ce dsquilibre avec des oprations locales

avec une complexit constante par noeud travers.

Mesurer le déséquilibre¶

Le dsquilibre d'un noeud est dfini partir de la hauteur de ses enfants droit et gauche

In [1]:
def equilibre(R):    
    if R == None: return 0
    else: return hauteur(R.gauche) - hauteur(R.droite)

Calculer cette hauteur peut se faire rcursivement

In [2]:
def hauteur(R):
    if R == None: return 0
    return max(hauteur(R.gauche),
               hauteur(R.droite)) + 1

Mais avec une complexit $\Theta(n)$ avec $n$ le nombre d'lments dans le sous-arbre dont R est la racine.

Pour rendre ce calcul plus efficace, on stocke la hauteur comme un attribut supplmentaire de chaque noeud

In [3]:
class Node:
    def __init__(self,val):
        self.data = val
        self.droite = None
        self.gauche = None
        self.hauteur = 1

La fonction hauteur a maintenant une complexit $\Theta(1)$

In [4]:
def hauteur(R):
    return R.hauteur if R else 0

Mais il faut maintenir la valeur de l'attribut jour lors de toute opration susceptible de le modifier (insertion, suppression)

In [5]:
def calculer_hauteur(R):
    if R:
        R.hauteur = max( hauteur(R.gauche), 
                        hauteur(R.droite)) + 1

Corriger le déséquilibre¶

L'opration de base pour corriger un dsquilibre est la rotation, gauche ou droite. Prenons par exemple l'arbre suivant

In [6]:
R = X = Node('X');        
A = X.gauche = Node('A'); Y = X.droite = Node('Y'); 
B = Y.gauche = Node('B'); C = Y.droite = Node('C')
import include.helpers as h; h.afficher_arbre_binaire(X)

Pour un arbre binaire de recherche, A < X < B < Y < C.

La rotation gauche consiste faire de Y - l'enfant droite de X - la racine, tout en maintenant la proprit d'arbre de recherche A < X < B < Y < C.

In [7]:
def rotation_gauche(x):
    y = x.droite
    x.droite = y.gauche
    y.gauche = x
    return y
In [8]:
R = rotation_gauche(R)

La rotation droite est l'opration inverse

In [9]:
def rotation_droite(x):
    y = x.gauche; x.gauche = y.droite; y.droite = x
    return y
In [10]:
R = rotation_droite(R)
h.afficher_arbre_binaire(R)

Quel est l'effet de ces rotations sur les hauteurs

  • des noeuds X et Y?
  • des sous-arbres de racine A, B et C?

Pour A, B et C, la rotation n'a aucun effet. Pour les noeuds X et Y, il faudra le recalculer. Les fonctions de rotations sont donc

In [11]:
def rotation_droite(x):
    y = x.gauche; x.gauche = y.droite; y.droite = x
    calculer_hauteur(x); calculer_hauteur(y)
    return y

L'opration complte d'quilibrage d'un noeud R s'crit donc

In [25]:
def equilibrer(R):
    assert(R != None)
    eq = equilibre(R)
    if eq < -1:
        if equilibre(R.droite) > 0:
            R.droite = rotation_droite( R.droite )
        R = rotation_gauche(R)    
    elif eq > 1:
        if equilibre(R.gauche) < 0:
            R.gauche = rotation_gauche(R.gauche)
        R = rotation_droite(R)
    else:
        calculer_hauteur(R)
    return R

Restent dterminer

  • quand appliquer cet quilibrage ?
  • quels noeuds l'appliquer ?
  • dans quel ordre traiter ces noeuds ?

La rponse ces questions est

  • Quand? - Aprs chaque opration qui peut modifier la hauteur des sous-arbres. Donc aprs insertion et suppression
  • Quels noeuds? - Tous les noeuds traverss par l'oprations, i.e. aux anctres des noeuds insrs ou supprims
  • Quel ordre? - Du noeud le plus profond la racine

Insertion¶

Le code de l'insertion est quasiment identique celui d'un arbre binaire de recherche. La seule diffrence est la dernire ligne qui retourne R quilibr.

Appeler equilibrer en retour de rcursion permet d'quilibrer les noeuds de la feuille la racine.

In [26]:
def inserer(R,val):
    if R == None:
        return Node(val)
    else:
        if val < R.data:
            R.gauche = inserer(R.gauche,val)
        elif val > R.data:
            R.droite = inserer(R.droite,val)
        else: # val == R.data
            pass
        return equilibrer(R)

def supprimer(R,val):
    if R == None: return R
    elif val < R.data: 
        R.gauche = supprimer(R.gauche,val)
    elif val > R.data:
        R.droite = supprimer(R.droite,val)
    else: # val == R.data
        if   R.gauche == None: return R.droite
        elif R.droite == None: return R.gauche
        else: # sous-arbres des deux cts
            tmp = plus_petit_noeud(R.droite)
            R.droite = supprimer_minimum(R.droite)
            tmp.gauche = R.gauche; tmp.droite = R.droite
            R = tmp
    return equilibrer(R)
