Tri rapide¶

Hoare, C. A. R. (1961). "Algorithm 64: Quicksort". Comm. ACM. 4 (7): 321

Principe¶

Le tri rapide est un algorithme rcursif consistant

  • choisir une valeur pivot parmi les lments du tableau
  • partitionner le tableau en: [ <= pivot ] pivot [ >= pivot ]
  • appeler rcursivement le tri rapide sur les 2 partitions

Partition¶

Entrée:¶

Le sous tableau T[premier:dernier] dont le dernier lment sert de pivot

Sortie:¶

L'indice i du pivot dans le tableau partitionn tel que

  • T[premier:i] contient les lments <= pivot
  • T[i] contient le pivot
  • T[i+1:dernier] contient les lments >= pivot

Algorithme: deux parcours simultanés de tableau¶

  • croissant (i), s'arrte quand T[i] >= pivot
  • dcroissant (j), s'arrte quand T[j] <= pivot

chaque arrt de i et j, on permute T[i] et T[j], puis on continue

Partition complète¶

  • effectuer les deux parcours
  • changer chaque double arrt

  • s'arrter quand les parcours se croisent.

  • placer le pivot la bonne place

In [9]:
def partition(T,premier,dernier):
    pivot = dernier-1
    i = premier; j = pivot-1
    
    while True:
        i = i_suivant(T,i,pivot)
        j = j_precedant(T,j,pivot)
        if j < i:
            break
        T[i],T[j] = T[j],T[i]
    
    T[i],T[pivot] = T[pivot],T[i]
    
    return i

Algorithme récursif¶

Cas trivial¶

un tableau de 0 ou 1 lment est dj tri.

Cas général¶

  • Choisir un pivot
  • Partitionner autour de ce pivot
  • trier rcursivement les deux cts de la partition

def tri_rapide_rec(T,premier,dernier):
    if premier < dernier-1:           # au moins deux lments
        pivot = dernier - 1                             # choix du pivot
        T[pivot],T[dernier-1] = T[dernier-1],T[pivot]   # dplacement du pivot en fin
        
        pivot = partition(T,premier,dernier)
        
        print("".rjust(premier*3),T[premier:pivot],T[pivot],T[pivot+1:dernier])
        
        tri_rapide_rec(T,premier,pivot)
        tri_rapide_rec(T,pivot+1,dernier)

Analyse théorique¶

Soit $C_n$ le nombre de comparaisons ncessaires pour le tri rapide de $n$ lments.

$C_n = (n+1) + C_k + C_{n-1-k}$

avec

  • $n+1$ le nombre de comparaisons pour la partition
  • $k$ le nombre d'lments gauche du pivot aprs partition
  • $n-k-1$ le nombre d'lments droite du pivot aprs partition

Avec un choix de pivot alatoire, toutes les valeurs de $k$ entre $0$ et $n-1$ sont quiprobables, de probabilit $\frac{1}{n}$.

$C_n$ vont donc en moyenne

$C_n = (n+1) + \sum_{k=0}^{n-1} \frac{1}{n}( C_k + C_{n-1-k} )$

et en notant que $\sum_{k=0}^{n-1} C_{n-1-k} = \sum_{u=0}^{n-1} C_{u}$ pour $u = n-1-k$, on obtient

$C_n = (n+1) + \frac{2}{n} \cdot \sum_{k=0}^{n-1} C_k$

Pour simplifier cette quation, notons que

$n \cdot C_n = n^2 + n + 2 \cdot \sum_{k=0}^{n-1} C_k $

$(n-1) \cdot C_{n-1} = n^2 - n + 2 \cdot \sum_{k=0}^{n-2} C_k $

En prenant la diffrence entre les deux, on obtient

$n \cdot C_n - (n-1) \cdot C_{n-1} = 2 \cdot n + 2 \cdot C_{n-1}$

En regroupant les termes et divisant par $n \dot (n+1)$, on trouve

$\frac{C_n}{n+1} = \frac{C_{n-1}}{n} + \frac{2}{n+1}$

en tenant compte de ce que $C_0 = C_1 = 0$ pour des tableaux de 0 ou 1 lments, on rsoud cet quation rcursive ce qui donne

$C_n = 2 \cdot (n+1) \cdot \sum_{k=3}^{n+1} \frac{1}{k}$

en approximant cette somme par une intgrale, on trouve

$C_n \approx 2 \cdot (n+1) \cdot \ln n$

en logarithme nprien, ce qui donne

$C_n \approx 1.39 \cdot n \cdot \log n$

en logarithme binaire.

Choix du pivot¶

Un choix alatoire du pivot n'est videmment pas optimal.

Le choix optimal serait de prendre la valeur mdiane, qui partitionne en deux parts gales. Mais trouver cette mdiane est trop cher

Un bon compromis consiste prendre la mdiane de 3 lments

In [31]:
def index_median(T,i1,i2,i3):
    if asd1.plus_petit(T[i1],T[i2]):
        if asd1.plus_petit(T[i2],T[i3]):
            return i2
        elif asd1.plus_petit(T[i1],T[i3]):
            return i3
        else:
            return i1
    else:
        if asd1.plus_petit(T[i1],T[i3]):
            return i1
        elif asd1.plus_petit(T[i2],T[i3]):
            return i3
        else:
            return i2

Stabilité¶

Le tri rapide n'est pas stable.

L'opration de partition dplace les lments selon leur comparaison avec le pivot et rien ne garantit que deux lments gaux restent ordonns pareillement.

In [33]:
asd1.test_stabilite(tri_rapide)

Le tri n'est pas stable
